## Module Biologie — nouveau
### Fichiers créés

### Tables SQL utilisées
- `C_ANALYSE` : catalogue des analyses (N°TypeAnalyse, analyse, rubrique)
- `LE_BILAN` : en-tête bilan (n_bilan, id, date_bilan) + colonne `observation` ajoutée
- `analyses` : détail (N_analyse, N_bilan, bilan=FK→C_ANALYSE, résultat)

### Script SQL exécuté
- `maj_c_analyse.sql` : vide et réinsère C_ANALYSE avec 54 analyses / 22 profils
- FK `FK_ANALYSES_CATALOGUE` désactivée pendant le DELETE puis réactivée
- Pas d'IDENTITY sur N°TypeAnalyse → numérotation manuelle 1→54

### Workflow biologie (4 colonnes)
```
[Col1 Profils] → [Col2 Analyses du profil] → [Col3 Panier] → [Col4 Résultats]
```
- Col1 : liste des profils + bouton ★ TOUS LES BILANS en bas (sticky)
- Col2 : analyses du profil cliqué, clic = ajout/retrait panier (✓/+)
- Col3 : boutons "Tout sélectionner" et "Insérer" EN HAUT, liste sélection dessous
- Col4 : tableau résultats sans rubrique, input vert si N, rouge si valeur anormale
- Barre du haut : sélecteur bilan, Nouveau, date, observation, N partout, Tout suppr, Imprimer
- Pas de doublons à l'insertion
- "TOUS LES BILANS" géré dynamiquement par PHP (get_all_analyses, distinct sur analyse)
